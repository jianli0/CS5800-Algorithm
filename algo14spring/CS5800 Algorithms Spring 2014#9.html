2. (25 points) A community has 3600 persons. On each day each person encounters 24 persons on average. A virus is introduced to the community by infecting one person. If a person is infected, the person remains infectious for 4 days on the average. The probability of an infectious person infecting another person during an encounter is the probability 0.1. Model this situation using a Markov chain in which the unit of time is one second. You may assume that at each second at most one person becomes infected and at most one person ceases to be infectious. Note that there is no immunity to the virus.

All that matters in this problem is the number of infected persons at each unit of time. So the state is the number of infected persons. There are 3601 states in all. The initial state is 1 infected person.

There are 3600 * 24 encounters per day on average, so there is 1 encounter per second on average. If the number of persons who are infected is n (i.e., the state is n), then the encounter will involve an infected person and an uninfected person with probability 2 * (n/3600) * (1-n/3600), and this will result in a new infection with probability 2 * (n/3600) * (1-n/3600) * 0.1 = (0.2 * n * (3600 - n)) / 12960000. In other words, there will be a transition from state n to state n+1 with this probability.

A person remains infected for t = 3600 * 24 * 4 = 345600 seconds on average. The probability that an infected person remains infected during each unit of time is t/(1+t), so the probability that an infected person ceases to be infected during each unit of time is 1/(1+t) = 1/345601. If there are n persons who are infected, then the probability that one of them ceases to be infected during each unit of time is n/345601. In other words, there will be a transition from state n to state n-1 with this probability.

The remaining transition is from state n to the same state. This occurs with the complementary probability to the other probabilities, so that all of them add up to 1.

4. (20 points) Modify the matrix in Question 3 using the Metropolis-Hastings method so that it has (0.25, 0.25, 0.25, 0.25) as an invariant distribution. Show your computation. Check that your modified matrix has this property.

The modified matrix is:
0.300.40.3
00.900.1
0.400.50.1
0.30.10.10.5

If this is your solution then you receive full credit. Otherwise, you need to show your work. For each i≠j, one needs to compute the value of α(i,j). If this is greater than or equal to 1, then the probability of transition from i to j does not change (i.e., it always happens). If α(i,j) is less than one, then transition from i to j only happens with probability α(i,j) so we need to multiply the probability of transition from i to j by this amount.

  1. α(1,2) = 0/0.2 = 0, so no transition occurs. The new probablity is 0.
  2. α(1,3) = 0.4/0.4 = 1, so probability does not change. It remains 0.4.
  3. α(1,4) = 0.5/0.3 > 1, so probability does not change. It remains 0.3.
  4. α(2,1) does not matter since this transition never occurs.
  5. α(2,3) = 0/0.7 = 0, so no transition occurs. The new probablity is 0.
  6. α(2,4) = 0.3/0.1 > 1, so probability does not change. It remains 0.1.
  7. α(3,1) = 0.4/0.4 = 1, so probability does not change. It remains 0.4.
  8. α(3,2) does not matter since this transition never occurs.
  9. α(3,4) = 0.1/0.2 = 0.5, so the probability changes from 0.2 to 0.1 = 0.2 * 0.5.
  10. α(4,1) = 0.3/0.5 = 0.6, so the probability changes from 0.5 to 0.3 = 0.5 * 0.6.
  11. α(4,2) = 0.1/0.3 = 1/3, so the probability changes from 0.3 to 0.1 = 0.3 * 1/3.
  12. α(4,3) = 0.2/0.1 > 1, so probability does not change. It remains 0.1.
The diagonal entries are determined by the requirement that the matrix be a stochastic matrix.

It is easy to check that (0.25,0.25,0.25,0.25) is invariant. One mistake that sometimes occurs is to assume that the objective of this problem is to find a matrix that has the specified distribution as an invariant distributions. The identity matrix for example, would be such a matrix. However, as the solution shows, the Metropolis-Hastings matrix is not the identity matrix in this case.

A stochastic matrix that has the uniform distribution as an invariant is called a doubly-stochastic matrix. They have the property that both the rows and the columns are probability distributions.

5. (10 points) In the game of cricket (or baseball) the players on a team take turns being the batsman (or batter in baseball) in a specified order. In baseball this is called the batting lineup. You have a program that will simulate a cricket match (or baseball game). The lineup order is considered to be an important strategic decision by a team.

Develop an algorithm using pseudo code for comparing one lineup with another one.

Note that one is comparing two lineups for the same team, not for different teams. The basic step of the algorithm is the following function that takes two lineups for the two teams and returns which team won the match: 

team playMatch(lineup A, lineup B) (1 point)
  Randomly play the game with team 1 with lineup A and team 2 with lineup B. (1 point)
  Return the winner. (1 point)

The following computes the number of times that a lineup for the first team will win against a random lineup for the second team: 

double playMatches(lineup A, int n) (1 point)
  set total to 0 (1 point)
  perform the following n times (1 point)
    randomly select a lineup B (1 point)
    total = total + playMatch(A, B) (1 point)
  return total / n (1 point)

One could also return total without dividing by n. Finally, let the two lineups to be compared be called L1 and L2. Run playMatches(L1, n) and playMatches(L2, n) using a large number n. The lineup with the significantly larger value is the better one. (1 point)
